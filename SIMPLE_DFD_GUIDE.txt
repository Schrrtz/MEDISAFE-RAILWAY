EXTERNAL ENTITIES (People/Systems that interact with your system)
================================================================================
□ Patient       - Users seeking medical services
□ Doctor        - Medical professionals providing care
□ Admin         - System administrators managing the platform


PROCESSES (Main functions of your system)
================================================================================
1.0  User Authentication
     - Login, Signup, Logout
     
2.0  Profile Management
     - View/Edit user profile and personal information
     
3.0  Appointment Booking
     - Book, view, cancel appointments with doctors
     
4.0  Live Consultation
     - Real-time video/text consultation with doctors
     
5.0  Prescription Management
     - Doctors create prescriptions, patients view them
     
6.0  Lab Results
     - Upload, view, and manage lab test results
     
7.0  Notifications
     - Send/receive system notifications and alerts
     
8.0  Admin Management
     - Manage users, doctors, patients, and system settings


DATA STORES (Database Tables)
================================================================================
D1  users              - User accounts (username, password, email, role)
D2  user_profiles      - Profile details (name, photo, address, contact)
D3  doctors            - Doctor information (specialization, license)
D4  patients           - Patient medical history
D5  appointments       - Scheduled appointments
D6  prescriptions      - Medical prescriptions
D7  lab_results        - Laboratory test results
D8  notifications      - System notifications


MAJOR DATA FLOWS (Connections between elements)
================================================================================

FROM PATIENT:
Patient → [1.0 Authentication] → D1 users
Patient → [2.0 Profile Mgmt] → D2 user_profiles
Patient → [3.0 Appointment Booking] → D5 appointments
Patient → [4.0 Live Consultation] → D3 doctors
Patient → [6.0 Lab Results] → D7 lab_results

FROM DOCTOR:
Doctor → [1.0 Authentication] → D1 users
Doctor → [2.0 Profile Mgmt] → D2 user_profiles
Doctor → [3.0 View Appointments] ← D5 appointments
Doctor → [4.0 Live Consultation] → Patient
Doctor → [5.0 Prescriptions] → D6 prescriptions

FROM ADMIN:
Admin → [1.0 Authentication] → D1 users
Admin → [8.0 Admin Management] → D1, D2, D3, D4

INTERNAL FLOWS:
[3.0 Booking] → [7.0 Notifications] → Patient/Doctor
[5.0 Prescriptions] → [7.0 Notifications] → Patient
[6.0 Lab Results] → [7.0 Notifications] → Patient
